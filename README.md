# monitoring-infra-qgis-plugins

> 생활인프라 편리성 모니터링을 위한 QGIS 플러그인 모음

시군구 단위로 공공시설의 **공급수준 · 향유수준 · 충족수준**을 산출하고,  
이를 부문별로 가중 합산한 **생활인프라 편리성 종합지수**를 계산합니다.

**제작**: 손종혁 (국토연구원 국토모니터링연구센터)

## 분석 체계

```
                ┌─ 공급수준 (1천인당 시설 수)
시설 데이터 ───┼─ 향유수준 (서비스권역 내 인구비율)   ──► 부문별 편리성 ──► 생활인프라
                └─ 충족수준 (충족 격자 비율)                                 편리성 종합지수
                                                                              (0 ~ 100점)
```

**5개 부문**: 교육학습 · 돌봄복지 · 보건의료 · 안전치안 · 체육문화

각 지표는 다음 3단계 전처리 파이프라인을 거칩니다.

```
원값 → [로그 변환] → 표준화 → 부문 내 평균
```

| 단계 | 선택지 |
|------|--------|
| 로그 변환 | `없음` / `ln(x+1)` / `log₁₀(x+1)` / `반사 ln` / `반사 log₁₀` |
| 표준화 | Min-Max / Z점수 / **T점수 (50+10Z)** / 백분위 / 없음 |

> 왜도(skewness) 통계를 자동 산출하여 최적 변환 방법을 권장합니다.  
> `|skew| > 2.0` → log₁₀, `|skew| > 1.0` → ln, 음의 왜도 → 반사 변환

## 플러그인 구성

| # | 폴더 | 플러그인명 | 버전 | 핵심 입력 |
|---|------|-----------|------|----------|
| 1 | `monitoring_qgis_plungin_supply_index_gpd` | 공급수준 분석 (GPD) | v1.0.4 | Point SHP + 시군구 인구 SHP |
| 2 | `monitoring_qgis_plungin_service_pop` | 향유수준 분석 | v1.0.0 | 서비스권역 SHP (value_r 컬럼) |
| 3 | `monitoring_qgis_plungin_access_idx` | 충족수준 분석 | v1.0.1 | 접근성 거리 SHP + 시군구 경계 SHP |
| 4 | `monitoring_qgis_plungin_composite` | 생활인프라 편리성 종합지수 | v1.0.0 | 위 1~3번 결과 SHP |

## 설치 방법

### 1. 플러그인 복사

각 플러그인 폴더를 QGIS 플러그인 디렉토리에 복사합니다.

```
# Windows
C:\Users\{사용자명}\AppData\Roaming\QGIS\QGIS3\profiles\default\python\plugins\

# macOS / Linux
~/.local/share/QGIS/QGIS3/profiles/default/python/plugins/
```

### 2. 의존성 설치

플러그인 1·2·3번은 `geopandas`를 사용합니다.  
**OSGeo4W Shell** (Windows) 또는 QGIS Python 콘솔에서 실행하세요.

```bash
pip install geopandas
```

### 3. QGIS에서 활성화

QGIS 재시작 → **플러그인 관리자** → 각 플러그인 체크 → 확인

## 플러그인별 상세

### 1. 공급수준 분석 (`monitoring_qgis_plungin_supply_index_gpd`)

시설 포인트 SHP와 시군구 인구 SHP를 공간 결합하여  
**시군구별 1천인당 시설 수**를 산출하고 부문별로 표준화합니다.

#### 입력 데이터
| 항목 | 형식 | 필수 컬럼 | 비고 |
|------|------|----------|------|
| Point SHP 폴더 | 포인트 SHP 복수 | — | 파일명에 시설명 포함 |
| 시군구 인구 SHP | 폴리곤 SHP | `sgg_cd`, 인구 컬럼 | 시군구 경계 + 인구 |

#### 분석 탭 구성
1. **데이터 입력** — SHP 폴더·파일 경로, 시군구 코드·인구 컬럼 설정
2. **부문 분류** — 시설별 부문(교육학습·돌봄복지 등) 수동 조정
3. **로그 변환** — 시설별 왜도 통계 확인 및 변환 방법 설정
4. **계산** — 표준화 방법 선택 후 실행

#### 출력
```
supply_index.shp
  {fac}_cnt   : 시설 개수
  {fac}_sup   : 1천인당 시설 수
  {fac}_std   : 표준화 값
  {sec}_avg   : 부문별 평균
```

---

### 2. 향유수준 분석 (`monitoring_qgis_plungin_service_pop`)

시군구 단위 서비스권역 내 인구비율(`value_r`)이 저장된 SHP를 읽어  
**부문별 향유수준**을 산출합니다. 공간 조인 없이 이미 집계된 값을 활용합니다.

#### 입력 데이터
| 항목 | 형식 | 필수 컬럼 | 비고 |
|------|------|----------|------|
| 서비스권역 SHP 폴더 | 폴리곤 SHP 복수 | `sgg_cd`, `value_r` | 파일명 패턴: `숫자.숫자 시설명(시군구) ...` |

#### 분석 탭 구성
1. **데이터 입력** — SHP 폴더 경로, 시군구 코드 컬럼 설정
2. **부문 분류** — 시설별 부문 수동 조정
3. **로그 변환** — **시설별** 왜도 통계 확인 및 변환 방법 개별 설정
4. **계산** — 표준화 방법 선택 후 실행

#### 출력
```
service_pop_index.shp
  {fac}       : value_r 원값
  {fac}_log   : 로그 변환값 (변환 적용 시)
  {fac}_std   : 표준화 값
  {sec}_avg   : 부문별 평균
```

---

### 3. 충족수준 분석 (`monitoring_qgis_plungin_access_idx`)

격자별 시설 접근성 거리를 사용자 정의 기준으로 이진화하여  
**시군구별 충족 격자 비율**을 집계한 뒤 부문별로 표준화합니다.

#### 입력 데이터
| 항목 | 형식 | 필수 컬럼 | 비고 |
|------|------|----------|------|
| 접근성 SHP 폴더 | 폴리곤 SHP 복수 | `gid`, `sgg_cd`, `value` | `value`: 접근성 거리(km) |
| 시군구 경계 SHP | 폴리곤 SHP | `sgg_cd` | 최종 집계 기준 |

#### 분석 탭 구성
1. **데이터 입력** — 경로 및 컬럼 설정
2. **부문 분류** — 시설별 부문 수동 조정
3. **거리 기준** — 시설별 충족 거리 기준(km) 설정 → **충족 격자 비율 산출** (백그라운드 처리)
4. **로그 변환** — **부문별** 충족 비율의 왜도 통계 확인 및 변환 방법 설정
5. **계산** — 표준화 방법 선택 후 실행

> **파이프라인 분리**: 거리 기준 확정 후 충족 격자 비율 집계(`compute_rat`)가 백그라운드로 실행되며,  
> 완료 후 실제 분포 통계를 로그 변환 탭에 표시합니다.

#### 출력
```
access_index.shp
  {sec}_ok    : 충족 격자 수
  {sec}_rat   : 충족 격자 비율
  {sec}_log   : 로그 변환값 (변환 적용 시)
  {sec}_std   : 표준화 값
```

---

### 4. 생활인프라 편리성 종합지수 (`monitoring_qgis_plungin_composite`)

공급·향유·충족수준 결과를 부문별로 가중 합산하여  
**0~100점 척도의 생활인프라 편리성 종합지수**를 산출합니다.

#### 입력 데이터
- 공급수준 결과: `supply_index.shp`
- 향유수준 결과: `service_pop_index.shp`
- 충족수준 결과: `access_index.shp`

#### 출력
```
composite_index.shp
  {sec}_conv  : 부문별 편리성 (교육학습·돌봄복지·보건의료·안전치안·체육문화)
  infra_raw   : 생활인프라 편리성 원값
  infra_idx   : 생활인프라 편리성 종합지수 (0 ~ 100)
```

## 분석 워크플로

```
① 원시 데이터 준비
   ├─ 시설 Point SHP (공급수준용)
   ├─ 서비스권역 SHP — value_r 컬럼 (향유수준용)
   └─ 접근성 거리 격자 SHP — value 컬럼 (충족수준용)

② 각 플러그인 실행 (순서 무관)
   ├─ 공급수준 → supply_index.shp
   ├─ 향유수준 → service_pop_index.shp
   └─ 충족수준 → access_index.shp

③ 종합지수 플러그인 실행
   └─ 위 3개 SHP 입력 → composite_index.shp

④ QGIS 레이어 시각화 / Streamlit 대시보드 연동
```
